PHASE 3: ENTITLEMENT AND UNLOCK LOGIC
================================================================================

Implement centralized entitlement logic.

Required behaviors:
- free users see only starter content unlocked
- Pro unlocks subscription-based features, not automatically all one-time packs unless bundled
- one-time packs permanently unlock their content
- feature upgrades unlock specific product capabilities
- bundles grant all included entitlements
- ownership state must be reflected across app screens immediately
- if a subscription lapses, recurring subscription entitlements deactivate but one-time owned products remain
- if a bundle includes a subscription and one-time products, handle each entitlement type correctly
- admin-granted entitlements must behave the same as purchased entitlements

Entitlement checks must be centralized in a service, store, or domain layer.
