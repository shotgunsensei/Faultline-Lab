1. Billing integration
Preferred approach:

- backend route or serverless function layer to verify purchases and write entitlements

Use the stack that fits Replit best while remaining maintainable.

2. Billing modes
Support two modes:
A. Production mode
- real Stripe products/prices
- real checkout
- real webhook fulfillment
- real entitlement updates

B. Development/demo mode
- mock checkout
- simulated purchases for testing
- clearly separated from production logic
- easy toggle via environment variable

3. Purchase flow requirements
Support:
- subscription checkout
- one-time purchase checkout
- upgrade from free to pro
- bundle purchase
- restore purchases / entitlement refresh
- ownership sync after payment
- proper success/cancel handling
- purchase history or entitlement history where practical

4. Billing safety requirements
- never trust the client alone for entitlements in production mode
- verify purchases server-side
- do not expose secret keys client-side
- handle idempotency for fulfillment
- gracefully handle webhook retries
- log billing failures clearly
